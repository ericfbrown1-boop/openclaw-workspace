## Dropbox OAuth Automation — Built ✅

**Problem:** Dropbox tokens expire every 4 hours, requiring manual renewal

**Solution:** Built OAuth 2.0 automation system with refresh tokens

**Files Created:**
- `dropbox-auth.py` - OAuth automation script (executable)
- `DROPBOX_AUTH_SETUP.md` - Detailed setup guide
- `DROPBOX_QUICKSTART.md` - 5-minute quick start guide
- Updated `dropbox-cli.py` - Auto-refresh integration
- Updated `.gitignore` - Exclude `.dropbox_auth.json` from git

**How It Works:**
1. One-time setup: Create Dropbox app, store credentials
2. One-time auth: Browser OAuth flow gets refresh token (never expires)
3. Auto-refresh: `dropbox-cli.py` checks token expiry before each call
4. If expired (<5 min left), automatically refreshes via `dropbox-auth.py`
5. Updates both `.dropbox_auth.json` and `dropbox-cli.py` with new token

**Commands:**
- Setup: `python3 dropbox-auth.py setup <app_key> <app_secret>`
- Authorize: `python3 dropbox-auth.py authorize` (opens browser)
- Status: `python3 dropbox-auth.py status`
- Manual refresh: `python3 dropbox-auth.py refresh`
- Get token: `python3 dropbox-auth.py token` (auto-refreshes if needed)

**Benefits:**
- Refresh token never expires (until manually revoked)
- Access tokens auto-renew seamlessly
- No more manual token copy/paste
- Secure storage with 0600 permissions
- Fallback to hardcoded token if auth system fails
