# Daily Notes — 2026-03-28

## FinancialReportApp — Post-Project Learnings (5:47 AM)

### The 12-Hour Arc
What started as "run a financial report for CrowdStrike" at ~9 PM turned into a full pipeline refactor that exposed 14 distinct bugs across 8 files. The app was generating reports marked "done" with zero content — the worst kind of bug: silent success with garbage output.

### Key Learnings (Ranked by Impact)

**1. Output Correctness > Pipeline Completion**
The original app checked if each step succeeded (crawl ✅, API call ✅, save ✅, email ✅) but never checked what the user actually received. A report with 100% placeholder text passed all checks. **Lesson: always open the deliverable and verify the content, not just the status.**

**2. Silent Fallbacks Are Bugs**
Every `except: return default_value` in the codebase was a silent failure. The parser returned `{"type": "other", "confidence": 0}` instead of raising, the report generator used "not available" instead of failing, the env validation didn't exist. **Lesson: fallbacks must log WARNING and count; >3 fallbacks = pipeline FAILURE.**

**3. Docker .env Inline Comments Are a Known Trap**
`ANTHROPIC_API_KEY=sk-ant-xxx # Required` truncated the key at `#`. Docker Compose issues #9025, #9327, #9509 document this. **Lesson: .env files must be KEY=value only, no inline comments, validate key length on startup.**

**4. Structured Outputs Eliminate Parsing Bugs**
Switching from free-text JSON parsing (regex + json-repair + bracket counting) to Anthropic `output_config/json_schema` eliminated an entire class of bugs. JSON is guaranteed valid by the API. **But** Anthropic's schema support is limited: no min/max on numbers, no minItems>1 on arrays. Enforce these in application code.

**5. Quality Gates Must Block, Not Log**
The first implementation logged "QUALITY WARNING" but still emailed the report. Gates that don't block aren't gates — they're decorations. **Lesson: if the gate fails, raise ValueError, delete the file, mark the task failed.**

**6. Don't Retry Content Failures**
A `ValueError` from a quality gate means the data is bad. Retrying with the same data produces the same bad result. Only retry transient failures (429, 500, timeout). **Lesson: catch ValueError separately from Exception in Celery tasks, return failure immediately.**

**7. Output Parity Across Delivery Channels**
The emailed .docx and the browser-downloaded file must be byte-identical. The fix: generate once → SHA256 hash → store hash → verify on download → explicit .docx MIME type on blob. **Lesson: generate once, hash, serve from single source, verify integrity on every delivery.**

**8. Parse the IR Homepage HTML**
Standard URL probing (`/press-releases`, `/investor-relations`) misses ASP.NET IR sites that use paths like `/financials/quarterly-results/default.aspx`. Extracting links from the actual IR homepage HTML found 16 pages where probing found 2 (8x improvement for Rubrik). **Lesson: always parse what the site actually contains, don't just guess URLs.**

**9. Dual-Model Review Catches What Single-Model Misses**
Grok 4.20 Beta's adversarial review found 8 issues that the original Opus implementation missed: toothless gates, missing retry logic, empty table rows, unsupported schema constraints. **Lesson: draft with one model, review with another. They have complementary blind spots.**

**10. Research Before Building**
The .env truncation fix was a 1-line change. But without knowing WHY (Docker Compose issue #9025), we would have just added quotes or tried other workarounds. One search found the permanent fix. **Lesson: 10 minutes of research beats 10 hours of workarounds.**
