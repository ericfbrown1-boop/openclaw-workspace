## Relevance Assessment

### 1. Claude Code Routines (research preview)
**HIGH** across most projects.
- **Jarvis pipeline**: Direct replacement for manual cron management — schedule Jarvis pipeline stages (ingest, enrich, report) as Routines with repo + connector access. **Action**: Prototype migrating at least one Jarvis nightly job to a Routine in Q2.
- **FinancialReportApp**: Could automate nightly financial data pulls + report generation without self-hosted cron infra. **Action**: Evaluate Routine for end-of-day report trigger.
- **Project Ajax**: Scheduled automation fits any CI/CD or agent loop tasks. **Action**: Identify 1-2 repetitive Ajax tasks to migrate.
- **Competitive Intel**: Schedule daily scrape/summarize routines. **Action**: High value — prototype immediately.
- **Salesforce/Snowflake analytics**: Low fit (data pipeline typically needs Snowflake-side scheduling), but connector support could help with summary push. **LOW-MEDIUM**.

### 2. Claude Code Desktop Redesign (parallel sessions + integrated terminal)
**MEDIUM-HIGH**.
- **FinancialReportApp / Project Ajax / Jarvis**: Parallel sessions let devs run multiple agents simultaneously — valuable for split-context debugging, frontend/backend parallel work. **Action**: Upgrade Claude app; explore parallel session workflow for next sprint.
- **Salesforce/Snowflake**: Integrated file editing + PDF preview useful for analytics report review. **MEDIUM**.

### 3. CHANGELOG v2.1.108 — Prompt Cache 1-hr TTL + /recap
**HIGH** for cost & agent continuity.
- **All projects with long-running agents (Jarvis, Ajax, FinancialReportApp)**: 1-hr prompt cache TTL dramatically reduces token cost on repetitive pipelines. Set `ENABLE_PROMPT_CACHING_1H=1` in all agent environments. **Action**: Add env var to all project `.env` / deployment configs immediately.
- **Jarvis pipeline**: `/recap` feature means returning agents get context summary — reduces context-window bloat on long sessions. **Action**: Enable; test with Jarvis multi-turn sessions.
- Model invoking `/security-review` via Skill tool = agents can self-trigger security scans. **MEDIUM** for FinancialReportApp security posture.

### 4. CHANGELOG v2.1.105 — PreCompact hooks + Plugin monitors + Skill description cap
**MEDIUM**.
- **Jarvis pipeline**: `PreCompact` hook enables guarding against mid-session compaction losing critical state. **Action**: Add PreCompact hook to Jarvis agent to checkpoint before compaction.
- **Skills/MCP**: Skill description cap raised to 1,536 chars — allows richer skill documentation. Review all custom skills. **Action**: Audit skill descriptions.
- **Plugin monitors**: Background monitors auto-arm at session start — useful for persistent watchers (e.g., file change monitors in FinancialReportApp). **MEDIUM**.

### 5. Model Deprecation — Sonnet 4 & Opus 4 (retire June 15, 2026)
**HIGH urgency**.
- **All projects**: Any hardcoded `claude-sonnet-4-20250514` or `claude-opus-4-20250514` model IDs must be updated to `claude-sonnet-4-6` / `claude-opus-4-6` before June 15. **Action**: Audit all project configs, `.env` files, and API call sites. Set reminder for May 15 as migration deadline.

### 6. Project Glasswing / Claude Mythos Preview
**LOW** (invite-only, defensive cybersecurity focus).
- **FinancialReportApp security**: If app handles sensitive financial data, watch for Glasswing GA. **Action**: Monitor; request invite if applicable.
