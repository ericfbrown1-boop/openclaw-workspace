### 1. Advisor Tool ⭐⭐⭐ HIGH — FinancialReportApp

**What it is:** Pairs a fast executor model (Sonnet 4.6) with a high-intelligence advisor (Opus 4.6) that provides strategic guidance mid-generation. Bulk token generation happens at executor pricing, quality stays at advisor level.

**Beta header:** `anthropic-beta: advisor-tool-2026-03-01`

**Eric's relevance:**
- **FinancialReportApp** currently runs Opus for both the tagging loop (~15 pages × 2K tokens) and the synthesis call (8K output tokens) — that's the biggest cost driver
- With Advisor Tool: Sonnet executes both, Opus advises on strategic analysis sections only
- **Estimated savings: ~65% per report (~$1.85 per run)**
- Quality gates in `generate_report_docx` remain as the safety net

**Proposed action:** Implement Advisor Tool in `tagger.py` + `generate_report.py`
**Files changed:** `config.py`, `tagger.py`, `generate_report.py`, `test_tagger.py` (4 files)
**Effort:** ~2.5 hours
**Status:** 📋 PLANNED — detailed implementation plan written 2026-04-14

### 4. `ant` CLI ⭐⭐ MEDIUM — Jarvis pipeline management

**What it is:** New Anthropic CLI for the Claude API. Native Claude Code integration. Versioned API resources in YAML files (prompt configs tracked in git). Think `gh` CLI but for Claude.

**Eric's relevance:**
- The 6 AM briefing, competitive intel cron, and earnings analyzer all have prompts hard-coded in Python scripts
- With `ant`: prompts become versioned YAML files in git → can A/B test prompt changes, track what changed between report versions, roll back a prompt without code deployment
- Particularly useful for the synthesis prompt in `tagger.py` — currently 50+ lines of inline string

**Proposed action:** Migrate Jarvis cron prompts to `ant`-managed YAML configs
**Status:** 💡 IDEA — low urgency, clean-up value

### 5. Background Monitor for Plugins (Claude Code 2.1.105) ⭐⭐⭐ HIGH — Monitor Agent redesign

**What it is:** Claude Code plugins can now declare a `monitors` manifest key that auto-arms at session start. Background monitors run continuously without cron overhead.

**Eric's relevance:**
- Directly applicable to the Monitor Agent improvement plan (requested same session 2026-04-14)
- Current Monitor Agent: polls every 5 min via cron → 12-step sweep → expensive, misses bursts between sweeps
- With background monitors: Monitor skill becomes a Claude Code plugin with a native background monitor that watches `gateway.log` and `openclaw.log` in real time → auto-triggers RCA when error patterns appear with zero polling latency
- **This is the architectural foundation of the Monitor Agent v2 plan**

**Proposed action:** Redesign Monitor skill as a Claude Code plugin with background monitor + real-time log watching + automatic RCA trigger
**Status:** 📋 PLANNED — part of Monitor Agent improvement plan (2026-04-14)

### 6. PreCompact Hook (Claude Code 2.1.105) ⭐⭐ MEDIUM — Jarvis context management

**What it is:** Hooks can now block context compaction by exiting with code 2 or returning `{"decision": "block"}`.

**Eric's relevance:**
- Current pain point: compaction during long pipeline runs (e.g., 30-min CrowdStrike crawl) loses in-flight stage state
- A PreCompact hook can check `memory/pipeline-state.json` → if a pipeline stage is `status: "running"` → block compaction until the stage writes its output file
- Would prevent the class of failures where context is compacted mid-synthesis and the LLM loses all the tagged page data

**Proposed action:** Add PreCompact hook to Jarvis workspace that checks pipeline-state.json
**Status:** 💡 IDEA — medium priority
